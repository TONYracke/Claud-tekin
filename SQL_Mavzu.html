SQL: DDL, DML, JOIN, Agregatsiya, Subquery va EXPLAIN

Times New Roman, 14pt — 10 sahifali qo'llanma

1-sahifa. SQL nima? DDL va DML tushunchasi

SQL (Structured Query Language) — relyatsion ma'lumotlar bazalari bilan ishlash uchun mo'ljallangan tizim tili. SQL ikki asosiy bo'limga bo'linadi:

  • DDL (Data Definition Language) — ma'lumotlar tuzilmasini aniqlash tili. Jadval, indeks, ko'rinish (view) va sxemalarni yaratish, o'zgartirish yoki o'chirish uchun ishlatiladi.
  • DML (Data Manipulation Language) — ma'lumotlarni boshqarish tili. Jadvaldagi qatorlarni kiritish, yangilash, o'chirish va tanlash uchun ishlatiladi.

DDL operatorlari bajarilganda avtomatik ravishda COMMIT sodir bo'ladi (aksariyat MBBT'larda). DML operatorlari esa tranzaksiyalar doirasida ishlaydi — ya'ni COMMIT yoki ROLLBACK bilan boshqariladi.

2-sahifa. DDL: CREATE operatori

CREATE — yangi ob'ekt (jadval, ma'lumotlar bazasi, indeks, view) yaratadi.

Ma'lumotlar bazasi yaratish:

CREATE DATABASE kompaniya
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

Jadval yaratish:

CREATE TABLE xodimlar (
  id         INT           NOT NULL AUTO_INCREMENT,
  ism        VARCHAR(100)  NOT NULL,
  familiya   VARCHAR(100)  NOT NULL,
  email      VARCHAR(150)  UNIQUE NOT NULL,
  lavozim    VARCHAR(80),
  maosh      DECIMAL(12,2) DEFAULT 0.00,
  tug_sana   DATE,
  bo_lim_id  INT,
  PRIMARY KEY (id),
  FOREIGN KEY (bo_lim_id) REFERENCES bo_limlar(id)
    ON DELETE SET NULL
    ON UPDATE CASCADE
);

Bu misolda: NOT NULL, UNIQUE, DEFAULT, PRIMARY KEY, FOREIGN KEY cheklovlari ishlatilgan.

3-sahifa. DDL: ALTER va DROP operatorlari

ALTER — jadval tuzilmasini o'zgartirish

Mavjud jadvalga ustun qo'shish, o'chirish yoki o'zgartirish mumkin:

-- Ustun qo'shish
ALTER TABLE xodimlar
  ADD COLUMN telefon VARCHAR(20) AFTER email;

-- Ustun turini o'zgartirish
ALTER TABLE xodimlar
  MODIFY COLUMN maosh DECIMAL(15,2) NOT NULL DEFAULT 0.00;

-- Ustunni qayta nomlash
ALTER TABLE xodimlar
  RENAME COLUMN tug_sana TO tugilgan_sana;

-- Ustunni o'chirish
ALTER TABLE xodimlar
  DROP COLUMN telefon;

-- Cheklov qo'shish
ALTER TABLE xodimlar
  ADD CONSTRAINT chk_maosh CHECK (maosh >= 0);

DROP — ob'ektni butunlay o'chirish

-- Jadvalni o'chirish (barcha ma'lumotlar bilan)
DROP TABLE IF EXISTS vaqtli_jadval;

-- Ma'lumotlar bazasini o'chirish
DROP DATABASE IF EXISTS test_db;

-- Ko'rinishni o'chirish
DROP VIEW IF EXISTS faol_xodimlar;

-- Indeksni o'chirish
DROP INDEX idx_email ON xodimlar;

TRUNCATE TABLE ham DDL'ga kiradi: u jadvaldagi barcha qatorlarni tezda o'chiradi, lekin tuzilmani saqlab qoladi. 

TRUNCATE TABLE jurnal;

4-sahifa. DDL: Tiplar va Cheklovlar

Asosiy ma'lumot turlari (MySQL/PostgreSQL):

  • Butun sonlar: TINYINT, SMALLINT, INT, BIGINT
  • Kasr sonlar: DECIMAL(p,s), FLOAT, DOUBLE
  • Matn: CHAR(n), VARCHAR(n), TEXT, LONGTEXT
  • Sana/Vaqt: DATE, TIME, DATETIME, TIMESTAMP
  • Mantiqiy: BOOLEAN / TINYINT(1)
  • Ikkilik: BLOB, BYTEA (PostgreSQL)
  • JSON: JSON (MySQL 5.7+, PostgreSQL 9.2+)

Cheklovlar (Constraints):

  • NOT NULL — ustun bo'sh bo'lishi mumkin emas
  • UNIQUE — har bir qiymat takrorlanmasin
  • PRIMARY KEY — NOT NULL + UNIQUE (asosiy kalit)
  • FOREIGN KEY — boshqa jadvaldagi asosiy kalitga havola
  • CHECK — qiymat shartni qanoatlantirishini tekshiradi
  • DEFAULT — qiymat kiritilmasa standart qiymat ishlatiladi
  • AUTO_INCREMENT / SERIAL — avtomatik raqamlab berish
CREATE TABLE buyurtmalar (
  id          BIGINT        PRIMARY KEY AUTO_INCREMENT,
  mijoz_id    INT           NOT NULL,
  summa       DECIMAL(14,2) NOT NULL CHECK (summa > 0),
  holat       VARCHAR(20)   NOT NULL DEFAULT 'kutilmoqda',
  yaratilgan  TIMESTAMP     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (mijoz_id) REFERENCES mijozlar(id)
);

5-sahifa. DML: INSERT, UPDATE, DELETE

INSERT — qator qo'shish

-- Yagona qator
INSERT INTO xodimlar (ism, familiya, email, maosh, bo_lim_id)
VALUES ('Sardor', 'Rahimov', 'sardor@mail.uz', 5500000, 2);

-- Bir nechta qator bir vaqtda
INSERT INTO xodimlar (ism, familiya, email, maosh, bo_lim_id) VALUES
  ('Malika', 'Yusupova', 'malika@mail.uz', 6200000, 1),
  ('Jasur',  'Toshmatov', 'jasur@mail.uz', 4800000, 3);

-- Boshqa jadvaldan ko'chirib qo'shish
INSERT INTO arxiv_xodimlar
SELECT * FROM xodimlar WHERE maosh < 3000000;

UPDATE — qatorlarni yangilash

-- Bitta xodim maoshini oshirish
UPDATE xodimlar
SET maosh = maosh * 1.15
WHERE id = 7;

-- Bir nechta ustunni bir vaqtda yangilash
UPDATE xodimlar
SET lavozim = 'Senior Developer',
    maosh   = 9000000
WHERE email = 'sardor@mail.uz';

-- Shartli ommaviy yangilash
UPDATE xodimlar
SET maosh = maosh + 500000
WHERE bo_lim_id = 2 AND maosh < 7000000;

DELETE — qatorlarni o'chirish

-- Bitta qatorni o'chirish
DELETE FROM xodimlar WHERE id = 12;

-- Shartga mos barcha qatorlarni o'chirish
DELETE FROM jurnal
WHERE yaratilgan < DATE_SUB(NOW(), INTERVAL 1 YEAR);

6-sahifa. JOIN turlari

JOIN — ikki yoki undan ortiq jadvaldan ma'lumotlarni birlashtiradi. Asosiy turlari:

  • INNER JOIN — faqat ikki jadvalda ham mos keladigan qatorlar
  • LEFT JOIN — chap jadvalning barcha qatorlari + o'ng jadvaldan mos kelganlar
  • RIGHT JOIN — o'ng jadvalning barcha qatorlari + chap jadvaldan mos kelganlar
  • FULL OUTER JOIN — ikkala jadvalning barcha qatorlari
  • CROSS JOIN — dekart ko'paytma (har bir qatorni har bir qatorga)
  • SELF JOIN — jadvalning o'zi bilan birlashishi
-- INNER JOIN: xodim va bo'lim nomi birga
SELECT x.ism, x.familiya, b.nomi AS bolim
FROM xodimlar x
INNER JOIN bo_limlar b ON x.bo_lim_id = b.id;

-- LEFT JOIN: bo'limi bo'lmagan xodimlar ham ko'rinsin
SELECT x.ism, b.nomi AS bolim
FROM xodimlar x
LEFT JOIN bo_limlar b ON x.bo_lim_id = b.id;

-- FULL OUTER JOIN (PostgreSQL):
SELECT x.ism, b.nomi
FROM xodimlar x
FULL OUTER JOIN bo_limlar b ON x.bo_lim_id = b.id;

-- SELF JOIN: xodim va uning menejerini ko'rsatish
SELECT x.ism AS xodim, m.ism AS menejer
FROM xodimlar x
LEFT JOIN xodimlar m ON x.menejer_id = m.id;

-- Ko'p jadval bilan JOIN
SELECT x.ism, b.nomi AS bolim, l.nomi AS loyiha
FROM xodimlar x
JOIN bo_limlar b ON x.bo_lim_id = b.id
JOIN loyiha_xodim lx ON lx.xodim_id = x.id
JOIN loyihalar l ON l.id = lx.loyiha_id;

7-sahifa. Guruhlash va Agregatsiya

GROUP BY — qatorlarni belgilangan ustun(lar) bo'yicha guruhlaydi. Agregat funksiyalar har bir guruh uchun yagona natija qaytaradi.

Asosiy agregat funksiyalar:

  • COUNT(*) — qatorlar soni
  • SUM(ustun) — yig'indisi
  • AVG(ustun) — o'rtacha qiymati
  • MIN(ustun) / MAX(ustun) — minimum / maksimum
  • GROUP_CONCAT(ustun) — guruh qiymatlarini birlashtirish (MySQL)
  • STRING_AGG(ustun, ajratuvchi) — PostgreSQL analogi
-- Har bir bo'limdagi xodimlar soni va o'rtacha maosh
SELECT
  b.nomi        AS bolim,
  COUNT(x.id)   AS xodimlar_soni,
  AVG(x.maosh)  AS ortacha_maosh,
  MAX(x.maosh)  AS max_maosh,
  MIN(x.maosh)  AS min_maosh
FROM xodimlar x
JOIN bo_limlar b ON x.bo_lim_id = b.id
GROUP BY b.id, b.nomi;

-- HAVING: guruhga shart qo'yish (WHERE o'rniga)
SELECT bo_lim_id, AVG(maosh) AS ortacha
FROM xodimlar
GROUP BY bo_lim_id
HAVING AVG(maosh) > 5000000
ORDER BY ortacha DESC;

-- Bir nechta ustun bo'yicha guruhlash
SELECT lavozim, bo_lim_id, COUNT(*) AS soni
FROM xodimlar
GROUP BY lavozim, bo_lim_id
ORDER BY soni DESC;

Qoida: SELECT ro'yxatidagi agregatsiz ustunlar GROUP BY da bo'lishi shart.

8-sahifa. Subquery (Ichki so'rovlar)

Subquery — boshqa SQL so'rov ichida ishlatiladigan to'liq SQL so'rov. U SELECT, FROM, WHERE va HAVING qismlarida ishlatilishi mumkin.

WHERE ichida subquery:

-- O'rtacha maoshdan yuqori maosh oluvchi xodimlar
SELECT ism, familiya, maosh
FROM xodimlar
WHERE maosh > (SELECT AVG(maosh) FROM xodimlar);

-- Eng ko'p xodimga ega bo'lim nomi
SELECT nomi FROM bo_limlar
WHERE id = (
  SELECT bo_lim_id
  FROM xodimlar
  GROUP BY bo_lim_id
  ORDER BY COUNT(*) DESC
  LIMIT 1
);

IN / NOT IN bilan subquery:

-- Hech qanday loyihaga biriktirilmagan xodimlar
SELECT ism, familiya
FROM xodimlar
WHERE id NOT IN (SELECT xodim_id FROM loyiha_xodim);

FROM ichida subquery (derived table):

SELECT bolim, ortacha_maosh
FROM (
  SELECT bo_lim_id AS bolim, AVG(maosh) AS ortacha_maosh
  FROM xodimlar
  GROUP BY bo_lim_id
) AS guruh_malumot
WHERE ortacha_maosh > 6000000;

EXISTS bilan subquery:

-- Kamida bitta buyurtmasi bor mijozlar
SELECT ism, familiya FROM mijozlar m
WHERE EXISTS (
  SELECT 1 FROM buyurtmalar b
  WHERE b.mijoz_id = m.id
);

CTE (Common Table Expression) — subquery'ni chiroyli yozish:

WITH yuqori_maosh AS (
  SELECT * FROM xodimlar WHERE maosh > 8000000
)
SELECT ism, familiya, maosh FROM yuqori_maosh
ORDER BY maosh DESC;

9-sahifa. EXPLAIN orqali so'rov ishlashini tahlil qilish

EXPLAIN (MySQL) va EXPLAIN ANALYZE (PostgreSQL) — SQL so'rovining bajarish rejasini (query plan) ko'rsatadi. Bu so'rov nima sabab sekin ishlashini aniqlash va optimallashtirish uchun zarur.

Ishlatilishi:

-- MySQL
EXPLAIN SELECT x.ism, b.nomi
FROM xodimlar x
JOIN bo_limlar b ON x.bo_lim_id = b.id
WHERE x.maosh > 5000000;

-- PostgreSQL (haqiqiy bajarish vaqti bilan)
EXPLAIN ANALYZE
SELECT x.ism, b.nomi
FROM xodimlar x
JOIN bo_limlar b ON x.bo_lim_id = b.id
WHERE x.maosh > 5000000;

EXPLAIN natijasidagi asosiy ustunlar (MySQL):

  • id — so'rov bajarilish tartibi
  • select_type — SIMPLE, PRIMARY, SUBQUERY, DERIVED va h.k.
  • type — qanday skanerlash: ALL (to'liq skan — eng yomon), index, range, ref, eq_ref, const (eng yaxshi)
  • possible_keys — ishlatilishi mumkin bo'lgan indekslar
  • key — haqiqatda ishlatiladigan indeks
  • rows — ko'rib chiqilishi taxmin qilingan qatorlar soni
  • Extra — qo'shimcha ma'lumot: Using index, Using filesort, Using temporary

Indeks yaratib optimallashtirilishi:

-- Maosh ustuniga indeks qo'shgach EXPLAIN natijasi yaxshilanadi
CREATE INDEX idx_xodim_maosh ON xodimlar(maosh);

-- Kompozit indeks (JOIN uchun)
CREATE INDEX idx_xodim_bolim ON xodimlar(bo_lim_id, maosh);

-- Endi EXPLAIN qayta tekshiriladi
EXPLAIN SELECT x.ism, b.nomi
FROM xodimlar x
JOIN bo_limlar b ON x.bo_lim_id = b.id
WHERE x.maosh > 5000000;

Maslahat: type = ALL va rows katta bo'lsa, indeks qo'shish so'rovni ko'p marta tezlashtirishi mumkin. Using filesort va Using temporary ko'rinsangiz — ORDER BY va GROUP BY ustunlariga indeks qo'shishni ko'rib chiqing.

10-sahifa. Kompleks misol va Xulosa

Quyida barcha o'tilgan mavzularni birlashtiruvchi kompleks misol keltirilgan. Unda DDL bilan jadvallar yaratiladi, DML bilan ma'lumot kiritiladi, JOIN, agregatsiya, subquery va EXPLAIN birgalikda ishlatiladi. 

-- 1. DDL: Jadvallar
CREATE TABLE bo_limlar (
  id    INT         PRIMARY KEY AUTO_INCREMENT,
  nomi  VARCHAR(80) NOT NULL UNIQUE
);

CREATE TABLE xodimlar (
  id        INT           PRIMARY KEY AUTO_INCREMENT,
  ism       VARCHAR(100)  NOT NULL,
  familiya  VARCHAR(100)  NOT NULL,
  maosh     DECIMAL(12,2) NOT NULL CHECK (maosh >= 0),
  bo_lim_id INT,
  FOREIGN KEY (bo_lim_id) REFERENCES bo_limlar(id)
);

-- 2. DML: Ma'lumot kiritish
INSERT INTO bo_limlar (nomi) VALUES ('IT'), ('Moliya'), ('Marketing');
INSERT INTO xodimlar (ism, familiya, maosh, bo_lim_id) VALUES
  ('Ali',   'Valiyev',  7500000, 1),
  ('Barno', 'Qodirova', 9200000, 1),
  ('Vohid', 'Azimov',   5100000, 2),
  ('Dilnoza','Salimova', 8300000, 3),
  ('Eldor', 'Tursunov', 4700000, 2);

-- 3. JOIN + Agregatsiya: Bo'limlar bo'yicha statistika
SELECT
  b.nomi                  AS bolim,
  COUNT(x.id)             AS xodimlar_soni,
  ROUND(AVG(x.maosh), 0)  AS ortacha_maosh,
  MAX(x.maosh)            AS eng_yuqori
FROM bo_limlar b
LEFT JOIN xodimlar x ON x.bo_lim_id = b.id
GROUP BY b.id, b.nomi
HAVING COUNT(x.id) > 0
ORDER BY ortacha_maosh DESC;

-- 4. Subquery: O'rtachadan yuqori maoshli xodimlar
SELECT ism, familiya, maosh,
  (SELECT nomi FROM bo_limlar WHERE id = x.bo_lim_id) AS bolim
FROM xodimlar x
WHERE maosh > (SELECT AVG(maosh) FROM xodimlar)
ORDER BY maosh DESC;

-- 5. EXPLAIN: So'rov rejasini tekshirish
EXPLAIN SELECT x.ism, b.nomi, x.maosh
FROM xodimlar x
JOIN bo_limlar b ON x.bo_lim_id = b.id
WHERE x.maosh > 6000000;
